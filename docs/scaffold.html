How to Scaffold a Project

1. Connect Remotely

Use SSH to remotely log into the server.

$ ssh yourname@142.93.199.169

You may need to approve an identity exception, and then you'll be asked to enter your password.

2. Navigate to your Directory

Note: You can always see the contents of a directory by using the list command with the -all and -long arguments, like this:

$ ls -al

You can nagivate to a location using the change directory command cd and entering the path to the directory you want to move to. You can use .. to move up a level.

$ cd ..
$ cd var/www/html/your_id

Finally, you will need to use the sudo command for much of this, but the command has been ommitted for brevity's sake. If something fails to work, make sure to use sudo.

3. Create Subdirectories

Use the "make directory" command to create your project structure.

$ mkdir proj1
$ mkdir proj1/images
$ mkdir proj1/styles
$ mkdir proj1/scripts
$ mkdir proj1/posts
$ mkdir proj1/comments

Or, you could perform the above commands more directly, like so:

$ mkdir proj1
$ cd proj1
$ mkdir images styles scripts posts comments

4. Create Default Files

First, add a stylesheet in the proper directory.

$ touch proj1/styles/style.css

Then, add a single rule to test it's working. For simple edits, it makes sense to use an in-built editor, such as nano.

$ nano proj1/styles/style.css

Once you've finished: exit, save, and overwrite.

Next, create the index file and edit it in the same way. Be sure to include some placeholder content as well as a link to your stylesheet (at its correct location!) to make sure it's all working.

5. Sync to a Git Repo

Log into http://github.com and create a new repository named proj1 or the like.

Then, start a git process listening on your directory.

$ git init

Next, hook up the two repos you just made.

$ git remote add origin https://github.com/yourname/project.git

You'll be prompted to tell github who you are. You'll need your github email address and your github password to proceed. Once you've entered them, you're safe to stage and commit your changes.

6. Stage and Commit your Changes

You can stage all changes at once using the . argument.

$ git add .

You can combine the comment with the commit.

$ git commit -m "First message."

Finally, push all the commits up to Github.

$ git push -u origin master
